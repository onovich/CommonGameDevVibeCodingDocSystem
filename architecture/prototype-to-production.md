# 原型到生产架构的收敛路径

**适用场景**: refactor

## 适用阶段

- 玩法已经跑通
- 配置、对象池、基础资源已存在
- 代码开始出现单例、引用袋、上帝类、分散 Update、逻辑层直改 UI
- 目标不是"继续堆功能"，而是低 churn 收敛到生产架构

## 核心原则

### 不是推倒重来

原型阶段的很多决策是正确的：
- 配置驱动
- 对象池
- 资源管理

重构的目标是改善结构，而不是否定原型。

### 渐进式收敛

一次只做一个 slice：
- 只搬运一个职责
- 只涉及少量文件
- 改完立即验证
- 验证通过后停止

## 推荐收敛顺序

### 1. 收口入口与依赖

**目标**：把启动、注入、绑定、生命周期入口集中到一个地方。

**先做什么**：
- 把散落在多个入口的初始化顺序收回组合根
- 让依赖关系显式化

**不要顺手做**：
- 不要在这一步拆大量业务逻辑
- 不要在这一步重写 UI 细节

### 2. 提炼共享上下文

**目标**：把"无结构引用袋"收敛成明确的上下文对象。

**先做什么**：
- 明确哪些对象是系统级共享依赖
- 集中放入 Context
- 把 Clear 对称清理做好

**不要顺手做**：
- 不要把所有东西都塞进 Context
- 不要把临时局部变量也提升成共享状态

### 3. 集中帧驱动

**目标**：避免多处 Update/FixedUpdate/LateUpdate 分散业务。

**先做什么**：
- 把业务驱动入口收口到系统层
- 明确每帧调用顺序

**不要顺手做**：
- 不要在这一步重构全部业务逻辑内部实现
- 不要把系统层重新做成上帝类

### 4. 下沉业务逻辑

**目标**：把跨对象规则从 Manager/Controller 中剥离出来。

**先做什么**：
- 一次只搬一个职责
- 让业务层以 Context 为主要输入
- 优先处理跨对象逻辑

**不要顺手做**：
- 不要把 UI 构建逻辑也搬进业务层
- 不要把资源加载逻辑混进业务层

### 5. 收口对象管理

**目标**：禁止在业务逻辑里散落对象创建、销毁、回收、注册。

**先做什么**：
- 创建入口先收口到 Factory
- 活动对象注册先收口到 Repo
- 回收路径收口到 Pool/Factory 边界

**不要顺手做**：
- 不要在这一步顺便大改所有实体字段
- 不要一边迁移一边更改玩法规则

### 6. 收口 UI 边界

**目标**：逻辑层可以调用 UI 边界，但不直接改具体 UI 组件。

**先做什么**：
- 先把"逻辑层直改具体 UI 组件"的地方改为经过 UI 边界
- 保持逻辑层依赖的是 UI 边界，而不是具体面板类

**不要顺手做**：
- 不要重新引入逻辑 -> UI 事件转发，只为了绕一层
- 不要让业务层直接依赖具体 UI 组件类型

### 7. 收口资源加载

**目标**：避免宽泛资源扫描和配置读取分散。

**先做什么**：
- 把配置读取入口收口到资源基础设施层
- 保持定向缓存/定向读取

**不要顺手做**：
- 不要恢复宽泛扫描
- 不要把 legacy 配置和当前配置继续混用后再做全量扫描

## 常见原型反模式

| 反模式 | 症状 | 优先落点 | 第一步安全操作 |
|--------|------|----------|----------------|
| 单例/全局引用袋 | 谁都能拿对象，依赖关系不清 | 组合根 + Context | 先把创建顺序和共享依赖列清 |
| 上帝类 Manager | 生成、波次、战斗、UI、回收都塞一起 | System + Domain + Factory | 先拆出一个最独立职责 |
| 逻辑层直改 UI | 业务类直接摸 UI 组件 | UI 边界层 | 先补一个 UI API，再迁一个调用点 |
| 分散对象创建/回收 | 各处创建/销毁/回池 | Factory + Pool + Repo | 先收一个对象类型的创建路径 |
| 静态活动列表 | 全局 List 到处读写 | Repo 注册表 | 先把 register/unregister 收口 |
| 配置宽泛扫描 | LoadAll、隐式命名 | 资源基础设施 | 先改成已知资产名定向读取 |

## 完成判定

可以认为一个原型模块已经基本收敛到生产架构，当它满足：

- 入口在组合根收口
- 共享依赖在 Context 收口
- 帧驱动集中在 System
- 跨对象业务在 Domain
- 创建/回收/注册路径收口
- UI 写操作经过 UI 边界
- 配置读取经过资源基础设施
- 每一步都经过验证
- 没有为了"解耦"而额外引入绕路层
- 没有恢复宽扫、散落 Update、散落创建/销毁
