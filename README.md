# Common Game Dev Vibe Coding Doc System

跨项目通用知识库，适用于游戏开发、架构设计、AI 辅助开发工作流。

## 目录结构

```
GlobalDocs/
├── workflow/           # 通用工作流与方法论
├── unity/             # Unity 引擎专属知识
├── architecture/      # 通用架构设计
├── project-types/     # 按项目类型分类的知识
├── skills/            # 可复用 skill 模板
└── tools/             # 工具与自动化
```

## 使用方式

### 在项目中引用全局知识

项目本地文档应只保留：
- 项目特有约束
- 当前架构锚点
- 本地资源路径
- 历史遗留问题

通用知识应引用此库，避免重复维护。

### 适用场景标记

文档按适用场景分类：
- **prototype** - 快速 demo / 原型阶段
- **production** - 完整架构项目
- **refactor** - 重构与架构收敛
- **universal** - 全阶段通用

## 维护原则

- 脱敏：不包含项目特定路径、命名、私有信息
- 通用：可跨项目复用
- 分层：按引擎、项目类型、阶段分类
- 精简：避免冗余，一个主题一份文档
